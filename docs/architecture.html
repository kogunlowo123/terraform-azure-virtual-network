terraform-azure-virtual-network

Executive Summary

This Terraform module deploys a comprehensive Azure Virtual Network infrastructure with multiple address spaces, subnets with service endpoint and delegation support, per-subnet Network Security Groups with flexible rule definitions, NAT Gateway with configurable public IPs, DDoS Protection plan association, NSG flow logs with Log Analytics traffic analytics, Private DNS zones with multi-VNet linking, and custom DNS server configuration. The module serves as the foundational networking layer for Azure deployments with submodules for standalone subnets, private endpoints, and VNet peering.

Component Breakdown

Virtual Network

Core network with multiple address spaces (CIDR blocks), custom DNS servers (falls back to Azure-provided DNS), and optional DDoS Protection Plan association. All resources are tagged with common tags.

Subnets

Map-based subnet configuration with address prefixes, optional service endpoints (e.g., Microsoft.Storage, Microsoft.Sql), and delegations for PaaS service integration (e.g., Microsoft.Web/serverFarms). Each subnet can have independent NSG rules.

Network Security Groups

Per-subnet NSGs with flexible rule definitions. Each rule supports priority, direction (Inbound/Outbound), access (Allow/Deny), protocol (Tcp/Udp/Icmp/*), port ranges, and address prefixes. NSGs are automatically associated with their subnets.

NAT Gateway

Optional NAT Gateway with configurable public IP count (1+), Standard SKU, and idle timeout. Automatically associated with all subnets for consistent outbound connectivity.

NSG Flow Logs

Optional flow logging per NSG with dedicated storage account (auto-created), version 2 format, 30-day retention, and traffic analytics integration with Log Analytics Workspace at 10-minute intervals.

Private DNS Zones

DNS zones for private endpoint resolution (e.g., privatelink.blob.core.windows.net). Auto-linked to the module's VNet plus optional additional VNets. Supports hub-and-spoke DNS architectures.

Data Flow

  1. Resources are deployed into subnets with designated address prefixes
  2. Inbound and outbound traffic is filtered by per-subnet NSG rules
  3. Service endpoints provide direct access to Azure PaaS services over the backbone
  4. Subnet delegations enable PaaS service injection (e.g., App Service, Container Instances)
  5. NAT Gateway provides consistent outbound IP for all subnets
  6. NSG flow logs capture traffic metadata and forward to Log Analytics for analytics
  7. Private DNS zones resolve private endpoint FQDNs to private IPs
  8. VNet links connect DNS zones to VNets for name resolution
  9. DDoS Protection Plan provides L3/L4 attack mitigation

Industry Adaptation

Hub-and-Spoke

Deploy as hub VNet with private DNS zones linked to spoke VNets, NAT Gateway for shared outbound, centralized NSG flow logging, DDoS protection at hub level.

Microservices

Dedicated subnets per tier (web/app/data), service endpoints for storage and SQL, delegated subnets for App Service and Container Instances, strict NSG rules between tiers.

Compliance

DDoS protection enabled, NSG flow logs with traffic analytics, all subnets with deny-by-default NSG rules, private DNS for all PaaS endpoints, no public exposure.

Multi-Region

VNet peering submodule for cross-region connectivity, DNS zones with multi-VNet linking, NAT Gateway per region for local outbound, consistent NSG rules across regions.

Production Readiness Checklist

  • Plan IP address space to avoid overlaps with peered VNets
  • Define subnet segmentation by application tier and security zone
  • Configure NSG rules with deny-by-default approach
  • Enable DDoS Protection for internet-facing workloads
  • Deploy NAT Gateway for predictable outbound connectivity
  • Enable NSG flow logs with traffic analytics
  • Create private DNS zones for all required private endpoints
  • Configure service endpoints for PaaS service access
  • Use subnet delegations for PaaS service injection
  • Set up custom DNS servers if using hybrid connectivity
  • Plan for VNet peering with hub-and-spoke topology
  • Apply resource tags for governance and cost tracking

Deployment

# Initialize Terraform
terraform init

# Preview changes
terraform plan -out=tfplan

# Apply configuration
terraform apply tfplan

Prerequisites

  • Terraform >= 1.5.0
  • AzureRM provider >= 3.80.0
  • Existing Azure Resource Group
  • For flow logs: Network Watcher in the same region
  • For DDoS: existing DDoS Protection Plan
